hmdccondor.HMDCCondor module¶

HMDCCondor module provides most-commonly-used functions for interacting with HTCondor clusters: polling for job status, acquiring schedd objects, and submitting jobs according to the HMDC ClassAd schema. This replaces Condor.pm in the hmdc-admin repository.

hmdccondor.HMDCCondor.poll_thread(id, return_status, use_local_schedd)[source]¶

poll_thread is a function which when provided as an argument to apply_async() is executed in an independent thread. poll_thread creates a new HMDCPoller which runs until the job represented by id has a JobStatus equal to any number in the return_status array.

Parameters:
  • id (int) – HTCondor Job Id to poll for
  • return_status (list) – An array of JobStatus values to look for.
  • use_local_schedd (boolean) – If set to true, uses the schedd of the executing host. If false, finds the schedd whcih submitted the HTCondor job represented by id.
Returns:

(JobStatus,ClassAd) or (None,None) if no such job exists.
Return type:

tuple
Example: 
>>> import hmdccondor
>>> from hmdccondor.HMDCCondor import poll_thread 
>>> poll_thread(10, [1,2,3,4,5], True)
hmdccondor.HMDCCondor.poll_xpra_thread(out_txt)[source]¶

poll_xpra_thread is a function when provided as an argument to apply_async() is executed in an independent thread. poll_xpra_thread continues to read out_txt, which is the location of the Xpra startup log file, to determine if xpra has started within a job.

Parameters:out_txt (str) – Fully qualified path of Xpra startup log file
Returns:Xpra display number
Return type:int
Example: 
>>> from hmdccondor import poll_xpra_thread
>>> poll_xpra_thread('/nfs/home/E/esarmien/.HMDC/jobs/interactive/xstata-mp_14.0_174_201510271445990519/out.txt')
